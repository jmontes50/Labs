🧪 Laboratorio Git: Colaboración con Pull y Push

📘 Introducción

Este laboratorio está diseñado para que practiques la colaboración usando Git y GitHub con otra persona. A través de una serie de pasos prácticos, aprenderás cómo crear un repositorio, trabajar con ramas, hacer cambios y sincronizarlos correctamente usando los comandos pull y push. Es una experiencia cercana al trabajo real en equipo, y te permitirá mejorar tus habilidades técnicas y de comunicación.

Lo ideal es realizar este Laboratorio en duplas, considera preguntar a uno(a) de tus compañeros(as) ó a alguien que este interesado en aprender un poco de git

🧠 Requisitos y conceptos previos

  • Conocer qué es Git, y qué hacen los comandos init, add, commit, push, pull, clone.
  • Tener Git y una cuenta en GitHub.
  • Git debe estar configurado (git config --global user.name y user.email).

👥 Participantes

  • Estudiante 1: Creador del repositorio.
  • Estudiante 2: Colaborador.

🛠️ Parte 1: Crear el proyecto

1. Crea una carpeta, estos comandos son para la terminal pero no hay problema puedes hacer manualmente. Elije la carpeta que desees 

mkdir git-lab && cd git-lab

2. Inicializa Git, en la carpeta de trabajo que hayas elegido:

git init

3. Crea un archivo index.html:

<!DOCTYPE html>
<html><head><title>Mi sitio</title></head>
<body><h1>Hola mundo</h1></body>
</html>

4. Haz commit:

git add .
git commit -m "Primer commit con index.html"

5. Crea repo en GitHub y enlaza, pon la ruta de tu repo creada:

git remote add origin https://github.com/tuusuario/repositorio.git
git push -u origin main

🔗 Parte 1.5: Invitar colaborador

En GitHub, ve a Settings → Collaborators, invita a Estudiante 2, y espera que acepte.

🤝 Parte 2: Clonar y contribuir

1. Clona el repo:

git clone https://github.com/tuusuario/repositorio.git

2. Crea un nuevo archivo saludo.html:

<h2>Hola desde Estudiante 2</h2>

3. Commit y push:

git add saludo.html
git commit -m "Estudiante 2 agrega saludo"
git pull origin main
git push origin main

🔄 Parte 3: Estudiante 1 actualiza

1. Pull y verifica:

git pull origin main

2. Agrega a index.html:

<p>Cambios realizados por Estudiante 1</p>

3. Commit y push:

git add index.html
git commit -m "Actualización de Estudiante 1"
git pull origin main
git push origin main

🔄 Parte 4: Estudiante 2 recibe cambios

1. Pull:

git pull origin main

2. Verifica cambios en index.html.

🔄 Parte 5: Más interacción

1. Crear ideas.html:

<ul><li>Idea 1</li></ul>

2. Commit y push:

git add ideas.html
git commit -m "Ideas por Estudiante 2"
git push origin main

1. Modifica index.html:

<p>Segunda edición de Estudiante 1</p>

2. Pull antes de hacer push:

git pull origin main
git push origin main

🤔 Reflexión, piensa acerca de esto

  • ¿Qué pasa si haces push sin pull?
  • ¿Cómo se resuelve un conflicto?
  • ¿Qué aprendiste del flujo colaborativo?
